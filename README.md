# 🎮 ECF_Game


## サイト概要

### サイトテーマ

ゲームを通じたユーザー同士の交流や情報共有を目的としたコミュニティサービスです。
ユーザーはゲームに関する投稿やコメント、グループへの参加を通じて、同じ趣味を持つユーザーと交流することができます。


### テーマを選んだ理由

私はゲームが好きで、これまで様々なジャンルのゲームを遊んできました。

近年ではSNSやオンラインコミュニティの発展により、ゲーム攻略情報の共有や一緒に遊ぶユーザー探しなど、ゲームを楽しむ上でユーザー同士の交流が重要になっていると感じました。
しかし、情報収集・交流・コミュニティ作成などが複数のサービスに分散していることも多く、一つのサービス内でゲームに関する情報共有や交流を完結できる環境があれば、より快適にゲームを楽しめると考えました。

そこで、個人投稿だけでなく、ユーザー自身がグループを作成し、メンバー同士で交流できるコミュニティサービスを制作しました。
また、実際のサービス運営を想定し、参加申請・メンバー管理・管理者機能などの機能を実装することで、より実践的なWebアプリケーション開発を目指しました。


### ターゲットユーザー

- ゲームについて他のユーザーと情報交換したい人
- 同じゲームを楽しむ仲間を探している人
- 自分の知識やプレイ経験を共有したい人
- ゲームに関するコミュニティを作成し、メンバーと継続的に交流したい人

### 主な利用シーン

- プレイしているゲームについて、他ユーザーの投稿やコメントから情報を得たい時
- 自分のプレイ経験やゲームに関する情報を他ユーザーへ共有したい時
- 同じゲームを楽しむユーザーと交流するためのコミュニティを探したい時
- 参加しているグループ内で、投稿・イベント・お知らせを通じてメンバー同士の交流を深めたい時

## 工夫した点

### グループ管理機能

単純な投稿機能だけではなく、ユーザーがコミュニティを形成できるように、グループ作成・参加申請・メンバー管理などを実装しました。

### 権限管理

一般ユーザーと管理者で利用できる機能を分離し、それぞれ異なる権限で操作できる設計にしました。

### 論理削除設計

ユーザー、投稿、コメント、グループ関連データは物理削除ではなく論理削除を採用しています。  
削除日時・削除理由・削除実行者を保持し、管理者による確認・復元を可能にしています。  
また、親データの削除による関連データへの影響を管理するため、
連鎖削除・連鎖復元後の非表示状態も記録できる設計にしています。

### Active Storageによる画像管理

画像管理にはActive Storageを利用しています。
本番環境ではActive Storageの保存先としてCloudinaryを利用しています。

### Seedによる環境再現

初期データ作成にはseedを利用しています。  
単純なサンプルデータ投入ではなく、実際の機能検証を想定したテストデータを作成しています。

グループ機能では、オーナー・モデレーター・一般メンバーの権限構成、参加状態、公開状態などを再現しています。また、以下の状態を確認できるようにしています。

- グループ内投稿・コメント・イベント・お知らせの作成
- 固定表示データ
- 非公開データ
- 外部公開設定データ
- 論理削除データ
- ユーザー削除による関連データの連鎖削除状態
- 削除後復元時に非表示となるデータ

管理者情報は環境変数から取得し、環境ごとに安全に初期データを構築できるようにしています。
